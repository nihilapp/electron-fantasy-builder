# 의존성 분석 - Fantasy Builder

**최종 업데이트**: 2026-02-07  
**버전**: 1.0.0  
**분석 일자**: 2026-02-07

## 개요 (Overview)

이 문서는 Fantasy Builder 프로젝트에서 사용되는 모든 의존성에 대한 포괄적인 분석을 제공하며, 각 의존성의 목적, 선정 이유, 그리고 전체 아키텍처에 기여하는 방식을 설명합니다.

## 런타임 의존성 분석 (Runtime Dependencies Analysis)

### 핵심 프레임워크 의존성 (Core Framework Dependencies)

#### **`electron: ^40.2.1`**
- **목적**: 크로스 플랫폼 데스크톱 애플리케이션 프레임워크
- **선정 이유**: 
  - 데스크톱 애플리케이션을 위한 웹 기반 개발 가능
  - 네이티브 OS 통합 제공 (창 관리, 시스템 트레이, 파일 시스템 접근)
  - 크로스 플랫폼 지원 (Windows, macOS, Linux)
  - 우수한 TypeScript 지원을 갖춘 성숙한 생태계
- **프로젝트 내 사용**: 
  - 메인 프로세스 관리 및 윈도우 수명 주기(Lifecycle) 관리
  - 메인과 렌더러 간의 IPC (프로세스 간 통신)
  - 네이티브 모듈 통합 (SQLite 데이터베이스)

#### **`vue: ^3.5.27`**
- **목적**: 사용자 인터페이스 구축을 위한 프로그레시브 JavaScript 프레임워크
- **선정 이유**:
  - Composition API를 통한 더 나은 TypeScript 지원
  - 반응형 시스템을 통한 우수한 성능
  - 모듈식 설계에 적합한 컴포넌트 기반 아키텍처
  - 거대한 생태계와 강력한 커뮤니티 지원
- **프로젝트 내 사용**:
  - 모든 사용자 인터페이스를 위한 프론트엔드 UI 프레임워크
  - 반응형 상태 관리를 위한 Composition API 활용
  - SPA (Single Page Application) 아키텍처

#### **`vue-router: ^5.0.2`**
- **목적**: Vue.js 공식 라우팅 라이브러리
- **선정 이유**:
  - Composition API와 네이티브 Vue 3 통합
  - Electron 앱에 적합한 해시(Hash) 히스토리 모드 지원
  - 복잡한 UI 구조를 위한 중첩 라우트(Nested route) 지원
- **프로젝트 내 사용**:
  - 뷰 간 네비게이션 (프로젝트, 설정 등)
  - 프로젝트 상세 섹션을 위한 중첩 라우팅
  - Electron 호환성을 위한 해시 기반 라우팅 (`/#/project/123`)

#### **`pinia: ^3.0.4`**
- **목적**: Vue를 위한 상태 관리 라이브러리
- **선정 이유**:
  - 공식 Vue 상태 관리 솔루션
  - 탁월한 TypeScript 지원
  - Vuex보다 간단한 API와 더 나은 성능
  - 모듈식 스토어 설계
- **프로젝트 내 사용**:
  - 전역 상태 관리 (프로젝트 목록, 사용자 설정)
  - 기능별 스토어 (특성, 능력 등)
  - API와의 반응형 상태 동기화

### API 및 백엔드 의존성 (API and Backend Dependencies)

#### **`hono: ^4.11.8`**
- **목적**: 엣지(Edge) 런타임을 위한 초고속 웹 프레임워크
- **선정 이유**:
  - 가볍고 빠름
  - 탁월한 TypeScript 지원
  - 최신 미들웨어 아키텍처
  - Electron 내장 API 서버로 사용하기에 완벽함
- **프로젝트 내 사용**:
  - 포트 3456에서 실행되는 내부 HTTP 서버
  - 메인-렌더러 통신을 위한 RESTful API 엔드포인트
  - CORS, 에러 처리, DB 컨텍스트를 위한 미들웨어

#### **`@hono/node-server: ^1.19.9`**
- **목적**: Hono를 위한 Node.js 어댑터
- **선정 이유**: Node.js 환경(Electron 메인 프로세스)에서 Hono 실행 가능
- **프로젝트 내 사용**: Electron 메인 프로세스 내에서 Hono API 서버 호스팅

#### **`@hono/zod-validator: ^0.7.6`**
- **목적**: Hono용 Zod 검증 미들웨어
- **선정 이유**: Zod 스키마 검증을 Hono 라우트와 통합
- **프로젝트 내 사용**: API 요청/응답 데이터의 런타임 검증

### 데이터베이스 및 ORM 의존성 (Database and ORM Dependencies)

#### **`better-sqlite3: ^12.6.2`**
- **목적**: Node.js를 위한 가장 빠르고 간단한 SQLite3 바인딩
- **선정 이유**:
  - 동기식 API로 Electron 메인 프로세스 코드 단순화
  - 로컬 데이터베이스 작업에 대한 탁월한 성능
  - 전체 SQLite 기능 지원
  - Electron을 위한 네이티브 바이너리 리빌드 필요
- **프로젝트 내 사용**: 오프라인 기능을 위한 로컬 SQLite 데이터베이스

#### **`drizzle-orm: ^0.45.1`**
- **목적**: SQL 데이터베이스를 위한 TypeScript 우선 ORM
- **선정 이유**:
  - 타입 추론을 지원하는 탁월한 TypeScript 지원
  - 데이터베이스를 숨기지 않는 SQL 유사 구문
  - SQLite와 PostgreSQL 모두 지원
  - 우수한 마이그레이션 시스템
- **프로젝트 내 사용**:
  - 데이터베이스 스키마 정의 및 마이그레이션
  - 타입 안전한 데이터베이스 작업
  - 이중 데이터베이스 지원 (로컬 SQLite, 원격 PostgreSQL)

#### **`pg: ^8.18.0`**
- **목적**: Node.js용 PostgreSQL 클라이언트
- **선정 이유**: 전체 기능을 지원하는 공식 PostgreSQL 드라이버
- **프로젝트 내 사용**: 온라인 기능을 위한 원격 PostgreSQL 데이터베이스 연결

### 검증 및 스키마 의존성 (Validation and Schema Dependencies)

#### **`zod: ^4.3.6`**

- **선정 이유**:
  - API 데이터에 대한 런타임 타입 체크
  - 자동 TypeScript 타입 추론

- **프로젝트 내 사용**:
  - API 요청/응답 검증
  - 폼 데이터 검증
  - 데이터 변환 및 파싱
